This is a setup to easily run `cadabra` code and observe the results in a browser with `quarto`.

Notes containing the calculation results are written in `quarto`. Calculations are performed by `cadabra 2`. Results are saved as `.tex` files inside the `results` folder and are later included in the quarto website. The `tools` folder contains scripts for cleaning the TeX files produced by `cadabra 2` and watching `cdb` files for changes.

Dependencies are a mess as a result of the complicated `cadabra 2` installation process. This assumes that you installed `cadabra 2` with your system python installation (tested under a fresh compilation of `cadabra 2` in Ubuntu 25.10 that comes with Python 3.13.7, for the scripts you can refer to the `pyproject.toml` file). The `.venv` generated by `uv` in this project is for helper scripts only.

Each `cdb` file can output one or more `.tex` fragments.

My worflow is the following:

1. Launch quarto with `uv run quarto preview .`
2. In the terminal where you will run the watcher, set `QUARTO_CMD` so the script uses the same Quarto executable. For this setup, use `export QUARTO_CMD="uv run quarto"`.
3. Launch the watcher with `uv run python tools/watch_build.py`
4. After you edit either a `.qmd` file or a `.cdb` file, everything should update automatically. When a `.cdb` file changes, the watcher reruns Cadabra and then explicitly rerenders the dependent `.qmd` page with Quarto, so the browser preview updates even if a plain file timestamp change is ignored.

If Quarto is already available directly on your shell `PATH`, you can skip setting `QUARTO_CMD`. In that case the watcher defaults to `quarto render`.

For actually starting a calculation:

1. Create a `.qmd` file for the notes.
2. Create a new `.cdb` file for the calculations.
   - Link the `.cdb` file to the `.qmd` file by including it in the header, something like the following in the header

````{markdown}
cadabra-deps:
  - cadabra-file.cdb
  - cadabra-file-2.cdb
````

3. Edit the `.qmd` file with your notes and the `.cdb` file with your calculations. Each time you save the `.cdb` file, the `.qmd` file updates. A `.cdb` file can write one `.tex` file directly to `results/<cdb-stem>.tex`, but the preferred layout is to write several `.tex` files into `results/<cdb-stem>/`. The watcher cleans every `.tex` file it finds in that directory before rerendering the dependent page. For example, if your file is `tests/test_covariant_derivative.cdb`, the watcher will clean all `.tex` files in `results/test_covariant_derivative/`.

To produce several outputs from one `.cdb` file, you can do something like this:

```{cadabra}
from pathlib import Path

output_dir = Path("results/test_covariant_derivative")
output_dir.mkdir(parents=True, exist_ok=True)

latex_code = result._latex_()
with open(output_dir / "main_result.tex", "w") as f:
  f.write(latex_code + "\n")
```

The older single-file layout still works too:

```{cadabra}
latex_code = result._latex_()
with open("results/test_covariant_derivative.tex", "w") as f:
    f.write(latex_code + "\n")
```

4. In order to render a generated `.tex` file, you should tell quarto to do so, for example with a python code block. This also allows you to process further the raw `.tex` output. Note that some basic processing is already done by `tools/fix_cadabra_latex.py`.

````{markdown}
```{python}
#| echo: false
#| output: asis

from pathlib import Path

tex = Path("results/test_covariant_derivative/main_result.tex").read_text(encoding="utf-8").strip()

print("::: {.scroll-math}")
print("$$")
print(tex)
print("$$")
print(":::")
```
````

When you continue a calculation, simply edit the `.cdb` file, write intermediate results as separate `.tex` files under `results/<cdb-stem>/`, and include whichever ones you want in the corresponding `.qmd` page.

As a bonus, this also includes a bibliography file that renders references as you'd expect in theoretical physics: Numbered, and with an ArXiv link.
